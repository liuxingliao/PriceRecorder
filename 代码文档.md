## 1. 架构设计概述

### 1.1 整体架构思想

本项目采用 **MVVM (Model-View-ViewModel)** 架构模式，结合 SwiftData 进行数据持久化。这种架构选择基于以下考虑：

#### 设计思想
1. **声明式UI优先**: SwiftUI 提供了声明式的UI编程方式，与 MVVM 模式天然契合
2. **数据驱动**: 使用 SwiftData 的 `@Query` 和 `@Model` 实现数据的自动响应式更新
3. **单向数据流**: 数据从 Model 流向 View，用户操作通过 ViewModel 或直接操作 Model
4. **关注点分离**: 
   - Model 层负责数据定义和持久化
   - View 层负责UI展示和用户交互
   - Service 层负责业务逻辑（CSV处理、云同步等）

#### 技术选型理由

| 技术 | 选择理由 | 替代方案对比 |
|-----|---------|------------|
| SwiftUI | 声明式UI，代码简洁，与 SwiftData 集成好 | UIKit：更成熟但代码量大 |
| SwiftData | 原生 ORM，iOS 17+ 官方支持，自动 CloudKit 集成 | Core Data：功能强大但API复杂 |
| SwiftCharts | 原生图表库，性能好，API简洁 | 第三方图表库：功能多但依赖重 |

## 2. 数据模型层详解

### 2.1 ProductRecord.swift - 商品记录模型

#### 文件路径
`PriceRecorder/Models/ProductRecord.swift`

#### 设计思想

这个模型是整个应用的核心数据模型，设计时考虑了以下要点：

1. **完整性与灵活性平衡**: 包含了商品记录所需的所有字段，但大部分字段是可选的
2. **自动计算字段**: `unitPrice` 在初始化时自动计算，确保数据一致性
3. **时间戳管理**: 包含 `createTime` 和 `updateTime` 两个时间戳，便于数据同步和审计
4. **关系设计**: 使用 `merchantID` 手动关联商家，而不是 SwiftData 的 `@Relationship`，这样做的好处是：
   - 更灵活的数据迁移
   - 避免循环引用问题
   - 更容易实现数据导出导入

#### 核心代码详解

```swift
@Model
final class ProductRecord {
    // 使用 @Attribute(.unique) 确保 id 唯一性，这是 SwiftData 的主键约束
    @Attribute(.unique) var id: UUID
    
    // 商品名称 - 必填字段，使用 String 类型
    var name: String
    
    // 品牌 - 可选字段，保留但不再作为独立模型管理
    var brand: String?
    
    // 数量 - 使用 Double 类型支持小数（如 2.5 斤）
    var quantity: Double
    
    // 单位 - 必填，如 "个"、"斤"、"盒" 等
    var unit: String
    
    // 规格 - 可选，如 "500ml"、"1kg" 等
    var spec: String?
    
    // 总价 - 必填，用户输入的总价
    var totalPrice: Double
    
    // 单价 - 自动计算，只读属性（但作为存储字段保存）
    var unitPrice: Double
    
    // 商家ID - 手动关联到 Merchant 模型
    var merchantID: UUID
    
    // 购买时间 - 必填，默认为当前时间
    var purchaseDate: Date
    
    // 小票照片 - 可选，存储为 Data 类型
    var receiptPhoto: Data?
    
    // 备注 - 可选
    var notes: String?
    
    // 创建时间 - 自动设置
    var createTime: Date
    
    // 更新时间 - 每次修改都应更新
    var updateTime: Date
```

**初始化方法设计**:

```swift
init(
    id: UUID = UUID(),  // 默认生成新的 UUID
    name: String,
    brand: String? = nil,
    quantity: Double,
    unit: String,
    spec: String? = nil,
    totalPrice: Double,
    merchantID: UUID,
    purchaseDate: Date = Date(),  // 默认为当前时间
    receiptPhoto: Data? = nil,
    notes: String? = nil,
    createTime: Date = Date(),
    updateTime: Date = Date()
) {
    self.id = id
    self.name = name
    self.brand = brand
    self.quantity = quantity
    self.unit = unit
    self.spec = spec
    self.totalPrice = totalPrice
    
    // 单价自动计算 - 关键设计！
    // 使用三目运算符避免除以零错误
    self.unitPrice = quantity > 0 ? totalPrice / quantity : 0
    
    self.merchantID = merchantID
    self.purchaseDate = purchaseDate
    self.receiptPhoto = receiptPhoto
    self.notes = notes
    self.createTime = createTime
    self.updateTime = updateTime
}
```

#### 设计亮点

1. **单价自动计算**: 在初始化时自动计算单价，确保单价和总价的一致性
2. **防除零保护**: `quantity > 0 ? totalPrice / quantity : 0` 防止数量为0时崩溃
3. **合理的默认值**: 大部分可选参数都有合理的默认值，简化调用
4. **时间戳分离**: 创建时间和更新时间分开，便于追踪数据变更

### 3.1 CSVService.swift - CSV导入导出服务

#### 文件路径
`PriceRecorder/Services/CSVService.swift`

#### 设计思想

CSV 服务是数据迁移的核心，设计时考虑：

1. **单例模式**: 使用 `static let shared` 提供全局访问点
2. **完整格式**: 导出包含所有字段，便于数据备份
3. **容错处理**: 导入时跳过无效行，不中断整个导入过程
4. **标准格式**: 遵循 CSV 标准，正确处理引号和转义

#### 数据结构

```swift
// CSV 导出数据结构 - 扁平化设计，包含关联数据
struct CSVExportData {
    let productName: String
    let brand: String?
    let quantity: Double
    let unit: String
    let spec: String?
    let totalPrice: Double
    let unitPrice: Double
    let merchantName: String  // 直接存储商家名称，不是ID
    let merchantCategory: String?  // 直接存储分类名称
    let purchaseDate: Date
    let notes: String?
    let createTime: Date
    let updateTime: Date
}
```

**设计亮点**: 导出时直接使用商家名称和分类名称，而不是 ID，这样 CSV 文件更容易阅读，也便于在不同设备间迁移。

#### 核心方法详解

##### 1. CSV 导出方法

```swift
func exportCSV(data: [CSVExportData]) throws -> String {
    // 表头行 - 明确列出所有字段
    var csvString = "商品名称,品牌,数量,单位,规格,总价,单价,商家名称,商家分类,购买时间,备注,创建时间,更新时间\n"
    
    for item in data {
        // 每一行都进行字段转义
        let row = [
            escapeCSVField(item.productName),
            escapeCSVField(item.brand ?? ""),
            "\(item.quantity)",
            escapeCSVField(item.unit),
            escapeCSVField(item.spec ?? ""),
            "\(item.totalPrice)",
            "\(item.unitPrice)",
            escapeCSVField(item.merchantName),
            escapeCSVField(item.merchantCategory ?? ""),
            dateFormatter.string(from: item.purchaseDate),
            escapeCSVField(item.notes ?? ""),
            dateFormatter.string(from: item.createTime),
            dateFormatter.string(from: item.updateTime)
        ].joined(separator: ",")
        
        csvString.append(row + "\n")
    }
    
    return csvString
}
```

**设计要点**:
- 表头用中文，便于用户理解
- 每个字段都调用 `escapeCSVField` 进行转义
- 可选字段为空时传入空字符串

##### 2. CSV 字段转义方法

```swift
private func escapeCSVField(_ field: String) -> String {
    // 检查是否需要转义：包含逗号、双引号或换行符
    if field.contains(",") || field.contains("\"") || field.contains("\n") {
        // 将字段内的双引号转义为两个双引号
        let escaped = field.replacingOccurrences(of: "\"", with: "\"\"")
        // 用双引号包裹整个字段
        return "\"\(escaped)\""
    }
    // 不需要转义的字段直接返回
    return field
}
```

**设计亮点**:
- 严格遵循 CSV 标准 (RFC 4180)
- 正确处理三种需要转义的情况
- 转义逻辑封装在私有方法中，易于维护

##### 3. CSV 行解析方法（状态机实现）

```swift
private func parseCSVLine(_ line: String) -> [String] {
    var fields: [String] = []
    var currentField = ""
    var inQuotes = false  // 状态标记：是否在引号内
    var index = line.startIndex
    
    while index < line.endIndex {
        let char = line[index]
        
        if char == "\"" {
            if inQuotes {
                // 在引号内遇到引号，检查是否是转义引号
                let nextIndex = line.index(after: index)
                if nextIndex < line.endIndex && line[nextIndex] == "\"" {
                    // 是转义引号，添加一个引号到字段
                    currentField.append("\"")
                    index = nextIndex  // 跳过下一个引号
                } else {
                    // 不是转义引号，结束引号状态
                    inQuotes = false
                }
            } else {
                // 不在引号内遇到引号，开始引号状态
                inQuotes = true
            }
        } else if char == "," && !inQuotes {
            // 不在引号内的逗号是字段分隔符
            fields.append(currentField)
            currentField = ""
        } else {
            // 普通字符，添加到当前字段
            currentField.append(char)
        }
        
        index = line.index(after: index)
    }
    
    // 添加最后一个字段
    fields.append(currentField)
    return fields
}
```

**设计亮点**:
- 使用状态机模式处理 CSV 解析，逻辑清晰
- 正确处理引号内的逗号和换行符
- 正确处理转义引号（两个引号表示一个引号）

##### 4. CSV 解析方法

```swift
func parseCSV(_ csvString: String) throws -> [CSVExportData] {
    var results: [CSVExportData] = []
    let lines = csvString.components(separatedBy: "\n").filter { !$0.isEmpty }
    
    // 至少需要表头 + 一行数据
    guard lines.count > 1 else {
        throw NSError(domain: "CSVService", code: -1,
            userInfo: [NSLocalizedDescriptionKey: "CSV文件格式无效"])
    }
    
    // 跳过表头，从第二行开始解析
    for line in lines.dropFirst() {
        let fields = parseCSVLine(line)
        guard fields.count >= 13 else { continue }  // 字段不够，跳过这一行
        
        // 解析数字和日期字段，使用 guard let 安全解析
        guard let quantity = Double(fields[2]),
              let totalPrice = Double(fields[5]),
              let unitPrice = Double(fields[6]),
              let purchaseDate = dateFormatter.date(from: fields[9]),
              let createTime = dateFormatter.date(from: fields[11]),
              let updateTime = dateFormatter.date(from: fields[12]) else {
            continue  // 解析失败，跳过这一行
        }
        
        let data = CSVExportData(
            productName: fields[0],
            brand: fields[1].isEmpty ? nil : fields[1],
            quantity: quantity,
            unit: fields[3],
            spec: fields[4].isEmpty ? nil : fields[4],
            totalPrice: totalPrice,
            unitPrice: unitPrice,
            merchantName: fields[7],
            merchantCategory: fields[8].isEmpty ? nil : fields[8],
            purchaseDate: purchaseDate,
            notes: fields[10].isEmpty ? nil : fields[10],
            createTime: createTime,
            updateTime: updateTime
        )
        
        results.append(data)
    }
    
    return results
}
```

**设计要点**:
- **容错设计**: 使用 `continue` 跳过无效行，不影响其他数据导入
- **安全解析**: 使用 `guard let` 逐一解析，避免崩溃
- **空值处理**: 空字符串转换为 `nil`，保持数据一致性

#### 日期格式化器

```swift
private let dateFormatter: DateFormatter = {
    let formatter = DateFormatter()
    formatter.dateFormat = "yyyy-MM-dd HH:mm:ss"  // 固定格式，便于跨平台
    return formatter
}()
```

**设计选择**: 使用固定的日期格式，而不是用户当前地区的格式，这样 CSV 文件在不同地区和设备间都能正确解析。

### 3.2 CloudSyncService.swift - iCloud云同步服务

#### 文件路径
`PriceRecorder/Services/CloudSyncService.swift`

#### 设计思想

云同步服务采用预留接口的设计方式：

1. **单例模式**: 全局共享一个同步服务实例
2. **模拟实现**: 当前版本是模拟实现，预留了真实 CloudKit 接入的接口
3. **状态管理**: 使用 `@Published` 属性，UI 可以自动观察同步状态
4. **设置持久化**: 使用 `UserDefaults` 保存自动备份设置

#### 核心代码详解

```swift
class CloudSyncService: ObservableObject {
    // 单例模式 - 全局共享
    static let shared = CloudSyncService()
    
    // 使用 @Published 标记，UI 可以观察这些属性的变化
    @Published var isSyncing = false
    @Published var lastSyncDate: Date?
    @Published var autoBackupEnabled = true
    
    private let userDefaults = UserDefaults.standard
    private let lastSyncKey = "lastSyncDate"
    private let autoBackupKey = "autoBackupEnabled"
    
    // 私有初始化方法，确保单例
    private init() {
        // 从 UserDefaults 恢复设置
        self.lastSyncDate = userDefaults.object(forKey: lastSyncKey) as? Date
        self.autoBackupEnabled = userDefaults.bool(forKey: autoBackupKey)
    }
```

**设计亮点**:
- `ObservableObject` 协议：使服务可以被 SwiftUI 视图观察
- 私有初始化：确保只能通过 `shared` 访问
- 自动恢复设置：初始化时从 UserDefaults 读取之前的设置

#### 备份方法设计

```swift
func backupToCloud(modelContext: ModelContext, completion: @escaping (Bool, Error?) -> Void) {
    isSyncing = true  // 开始同步，UI 会显示进度
    
    // 模拟网络延迟 - 2秒后完成
    DispatchQueue.global(qos: .userInitiated).asyncAfter(deadline: .now() + 2) {
        DispatchQueue.main.async {  // 切回主线程更新 UI
            self.isSyncing = false
            self.updateLastSyncDate()
            completion(true, nil)  // 回调通知完成
        }
    }
}
```

**设计要点**:
- **异步设计**: 使用 `@escaping` 闭包回调，不阻塞主线程
- **线程安全**: 在后台队列执行耗时操作，完成后切回主线程更新 UI
- **状态更新**: 正确更新 `isSyncing` 状态，UI 会自动刷新

#### 自动备份触发逻辑

```swift
func triggerAutoBackupIfNeeded(modelContext: ModelContext) {
    guard autoBackupEnabled else { return }  // 自动备份未开启，直接返回
    
    let now = Date()
    if let lastSync = lastSyncDate {
        let timeInterval = now.timeIntervalSince(lastSync)
        if timeInterval < 3600 {  // 距离上次同步不到1小时，跳过
            return
        }
    }
    
    // 执行自动备份
    backupToCloud(modelContext: modelContext) { _, _ in }
}
```

**设计亮点**:
- **节流机制**: 距离上次同步不到1小时不执行，避免频繁备份
- **静默执行**: 自动备份不显示 UI，安静执行

```swift
private func importCSV(from url: URL) {
    guard url.startAccessingSecurityScopedResource() else {
        alertTitle = "导入失败"
        alertMessage = "无法访问文件"
        showingAlert = true
        return
    }
    defer { url.stopAccessingSecurityScopedResource() }
    
    do {
        let content = try String(contentsOf: url, encoding: .utf8)
        let csvData = try CSVService.shared.parseCSV(content)
        
        var successCount = 0
        // 关键：本地缓存字典
        var merchantCache: [String: Merchant] = [:]
        var categoryCache: [String: MerchantCategory] = [:]
        
        // 先填充缓存
        for merchant in merchants {
            merchantCache[merchant.name] = merchant
        }
        for category in categories {
            categoryCache[category.name] = category
        }
        
        for data in csvData {
            // 处理分类
            let category: MerchantCategory?
            if let categoryName = data.merchantCategory {
                if let existing = categoryCache[categoryName] {
                    category = existing
                } else {
                    // 创建新分类并加入缓存
                    let newCategory = MerchantCategory(name: categoryName)
                    modelContext.insert(newCategory)
                    categoryCache[categoryName] = newCategory
                    category = newCategory
                }
            } else {
                category = nil
            }
            
            // 处理商家
            let merchant: Merchant
            if let existing = merchantCache[data.merchantName] {
                merchant = existing
            } else {
                // 创建新商家并加入缓存
                merchant = Merchant(
                    name: data.merchantName,
                    categoryID: category?.id
                )
                modelContext.insert(merchant)
                merchantCache[data.merchantName] = merchant
            }
            
            // 创建商品记录
            let product = ProductRecord(
                name: data.productName,
                brand: data.brand,
                quantity: data.quantity,
                unit: data.unit,
                spec: data.spec,
                totalPrice: data.totalPrice,
                merchantID: merchant.id,
                purchaseDate: data.purchaseDate,
                notes: data.notes,
                createTime: data.createTime,
                updateTime: data.updateTime
            )
            modelContext.insert(product)
            successCount += 1
        }
        
        importResult = (successCount, csvData.count)
        alertTitle = "导入完成"
        alertMessage = "成功导入 \(successCount)/\(csvData.count) 条数据"
    } catch {
        alertTitle = "导入失败"
        alertMessage = error.localizedDescription
    }
    showingAlert = true
}
```

**问题根源分析**:

之前的实现有问题，因为：
```swift
// ❌ 错误做法：直接查询 @Query 的 merchants 数组
for data in csvData {
    if let existing = merchants.first(where: { $0.name == data.merchantName }) {
        // ...
    } else {
        let merchant = Merchant(...)
        modelContext.insert(merchant)
        // ❌ @Query 的 merchants 数组不会立即更新！
        // 下一次循环时 merchants.first 还是找不到！
    }
}
```

**解决方案设计**:

使用本地缓存字典：
```swift
// ✅ 正确做法：使用本地缓存字典
var merchantCache: [String: Merchant] = [:]
var categoryCache: [String: MerchantCategory] = [:]

// 1. 导入前先填充缓存
for merchant in merchants {
    merchantCache[merchant.name] = merchant
}

// 2. 导入时查询缓存，新创建的也加入缓存
for data in csvData {
    if let existing = merchantCache[data.merchantName] {
        merchant = existing
    } else {
        merchant = Merchant(...)
        modelContext.insert(merchant)
        merchantCache[data.merchantName] = merchant  // 关键：同步更新缓存
    }
}
```

**设计亮点**:

1. **字典查找 O(1)**: 使用字典查找比数组遍历 O(n) 快得多
2. **双重缓存**: 商家和分类都有缓存
3. **缓存同步**: 新创建的对象立即加入缓存
4. `defer`: `defer` 确保安全停止访问文件
5. **结果统计**: 显示成功/失败数量，给用户明确反馈

### 3. OCRService 优化

#### 图片压缩功能
添加图片压缩，减少内存占用和处理时间：

```swift
/// 最大图片尺寸（宽度或高度），用于压缩
private let maxImageDimension: CGFloat = 1024

/// 压缩图片到指定尺寸
private func compressImage(_ image: UIImage) -> UIImage {
    let width = image.size.width
    let height = image.size.height

    // 如果图片已经小于最大尺寸，直接返回
    guard width > maxImageDimension || height > maxImageDimension else {
        return image
    }

    // 计算缩放比例
    let scale = min(maxImageDimension / width, maxImageDimension / height)
    let newSize = CGSize(width: width * scale, height: height * scale)

    // 渲染压缩后的图片
    let renderer = UIGraphicsImageRenderer(size: newSize)
    let compressedImage = renderer.image { _ in
        image.draw(in: CGRect(origin: .zero, size: newSize))
    }

    return compressedImage
}
```

#### Async/Await 支持

```swift
/// 现代 async/await 版本的文字识别
func recognizeText(from image: UIImage) async throws -> [OCRResult] {
    let compressedImage = compressImage(image)

    guard let cgImage = compressedImage.cgImage else {
        throw NSError(domain: "OCRService", code: -1,
            userInfo: [NSLocalizedDescriptionKey: "无法获取图片数据"])
    }

    return try await withCheckedThrowingContinuation { continuation in
        let request = VNRecognizeTextRequest { request, error in
            if let error = error {
                continuation.resume(throwing: error)
                return
            }

            var results: [OCRResult] = []
            if let observations = request.results as? [VNRecognizedTextObservation] {
                for observation in observations {
                    if let topCandidate = observation.topCandidates(1).first {
                        let result = OCRResult(
                            text: topCandidate.string,
                            boundingBox: observation.boundingBox
                        )
                        results.append(result)
                    }
                }
            }

            continuation.resume(returning: results)
        }

        // ... 配置 request
        let handler = VNImageRequestHandler(cgImage: cgImage, options: [:])

        DispatchQueue.global(qos: .userInitiated).async {
            do {
                try handler.perform([request])
            } catch {
                continuation.resume(throwing: error)
            }
        }
    }
}
```

```swift
struct AIChatView: View {

    @Environment(\.modelContext) private var modelContext
    
    @Query private var llmConfigs: [LLMConfig]
    @Query private var products: [ProductRecord]

                        .textFieldStyle(.roundedBorder)

                        if isSending {
                            ProgressView()
                        } else {
                            Image(systemName: "paperplane.fill")
                                .foregroundColor(.blue)
                        }
                    }
                    .disabled(inputText.isEmpty || activeConfig == nil)
                }
                .padding()
            }
            .navigationTitle("AI咨询")
            .toolbar {
                ToolbarItem(placement: .navigationBarLeading) {

                    }

    private func sendMessage() {
        guard let config = activeConfig else { return }
        
        let userMessage = AIMessage(role: "user", content: inputText)
        messages.append(userMessage)
        let question = inputText
        inputText = ""
        isSending = true
        
        Task {
            do {
                let systemPrompt = buildSystemPrompt()
                let response = try await LLMService.shared.sendChat(
                    messages: [
                        ChatMessage(role: "system", content: systemPrompt),
                        ChatMessage(role: "user", content: question)
                    ],
                    config: config
                )
                
                await MainActor.run {
                    let assistantMessage = AIMessage(role: "assistant", content: response)
                    messages.append(assistantMessage)
                    isSending = false
                }
            } catch {
                await MainActor.run {
                    let errorMessage = AIMessage(role: "assistant", content: "抱歉，出错了: \(error.localizedDescription)")
                    messages.append(errorMessage)
                    isSending = false
                }
            }
        }
    }
    
    private func buildSystemPrompt() -> String {
        // 构建包含商品数据的系统提示词

        // 加入商品统计信息
        prompt += "总商品记录数: \(products.count)\n"
        
        // ... 加入更多数据
        
        return prompt
    }
```

```swift
        // 按分类统计，倒序排列
        var stats: [String: Int] = [:]
        for product in merchantProducts {
            let categoryName = ...
            stats[categoryName, default: 0] += 1
        }
        return stats.sorted { $0.value > $1.value }
    }
```
