## 项目状态

代码已创建完成，主要文件已就绪。由于这是一个完整的Xcode项目，需要在Xcode中进行配置和编译。

## 第一步：在Xcode中打开项目

1. 打开Xcode
2. 选择 `File` → `Open`
3. 选择 `PriceRecorder.xcodeproj` 文件
4. 等待项目加载完成

## 第二步：配置项目设置

### 2.1 配置签名和Team

1. 在Xcode左侧项目导航器中，点击最顶部的项目名称（蓝色图标）
2. 选择 `TARGETS` → `PriceRecorder`
3. 选择 `Signing & Capabilities` 标签页
4. 在 `Team` 下拉菜单中选择您的开发团队
5. 修改 `Bundle Identifier` 为唯一的标识符（例如：`com.yourname.PriceRecorder`）

### 2.2 确认部署目标

1. 在 `General` 标签页中
2. 确认 `Minimum Deployments` → `iOS` 设置为 `17.0` 或更高

## 第三步：添加测试Target（可选）

如果需要运行单元测试和UI测试，需要添加测试Target：

### 3.1 添加单元测试Target

1. 点击项目导航器中的项目名称
2. 点击 `TARGETS` 区域下方的 `+` 按钮
3. 选择 `iOS Unit Testing Bundle`
4. 产品名称：`PriceRecorderTests`
5. 点击 `Finish`
6. 将 `PriceRecorderTests/PriceRecorderTests.swift` 文件添加到这个Target中

### 3.2 添加UI测试Target

1. 同样点击 `+` 按钮
2. 选择 `iOS UI Testing Bundle`
3. 产品名称：`PriceRecorderUITests`
4. 点击 `Finish`
5. 将 `PriceRecorderUITests/PriceRecorderUITests.swift` 文件添加到这个Target中

## 第四步：编译项目

### 4.1 基本编译

1. 在Xcode顶部选择一个模拟器（例如：iPhone 15）
2. 点击 `Product` → `Build`（或按 `⌘B`）
3. 等待编译完成
4. 查看左侧的Issue Navigator，修复任何编译错误

### 4.2 常见编译错误及修复

#### 错误1：No such module 'SwiftData'
**原因**: 部署目标版本低于iOS 17.0
**修复**: 将部署目标设置为iOS 17.0或更高

#### 错误2：Cannot find 'EmptyStateView' in scope
**原因**: CommonComponents.swift没有正确添加到Target
**修复**: 
1. 选中CommonComponents.swift文件
2. 在右侧File Inspector中，确保Target Membership勾选了PriceRecorder

#### 错误3：Signing for "PriceRecorder" requires a development team
**原因**: 没有配置开发团队
**修复**: 按照第二步配置Team和Bundle Identifier

#### 错误4：Cannot assign to property: 'url' is a 'let' constant
**原因**: ModelConfiguration初始化方式错误
**修复**: 使用 `ModelConfiguration(url: storeURL)` 而不是修改url属性

#### 错误5：'self' used before all stored properties are initialized
**原因**: 在init中使用了实例属性
**修复**: 将相关属性改为静态属性

## 第五步：运行应用

1. 在Xcode顶部选择一个模拟器或连接真机
2. 点击运行按钮（▶️）或按 `⌘R`
3. 等待应用启动
4. 在模拟器/真机上测试功能

## 第六步：运行测试（可选）

### 6.1 运行所有测试

- 点击 `Product` → `Test`（或按 `⌘U`）

### 6.2 运行特定测试

- 在Xcode左侧打开Test Navigator（💎图标）
- 点击特定测试旁边的播放按钮

## 第七步：真机测试（可选）

### 7.1 配置真机

1. 用USB线连接iPhone到Mac
2. 在iPhone上信任这台电脑
3. 在Xcode顶部设备选择器中选择您的iPhone
4. 点击运行

### 7.2 配置权限

真机测试需要在Info.plist中配置的权限（已配置）：
- 相机权限：`NSCameraUsageDescription`（预留）
- 相册权限：`NSPhotoLibraryUsageDescription`（预留）

## 豆包录入测试

1. 在设置中配置豆包链接（可选，有默认值）
2. 在首页点击"商品录入"
3. 选择"豆包录入"
4. 点击"打开豆包"（需要网络连接）
5. 在豆包中复制商品JSON数据
6. 粘贴到JSON编辑器
7. 点击"解析商品"
8. 确认商品列表
9. 选择商家
10. 保存

## LLM配置和测试（v1.3新增）

1. 进入设置 → AI模型配置
2. 查看默认配置（如果没有会自动创建）
3. 点击"+"添加新配置
4. 选择提供商（豆包/OpenAI/Claude/通义千问/文心一言/其他）
5. 输入API Key
6. 确认Endpoint和Model（有默认值）
7. 点击"测试接口"按钮
8. 查看测试结果
9. 点击保存（可以用导航栏的保存按钮或底部的保存按钮）
10. 在配置列表中启用某个配置（会自动禁用其他配置）

## AI咨询功能测试（v1.3新增）

1. 首先配置并启用一个LLM配置
2. 进入搜索页面

4. 在弹窗中输入问题（例如："我最近买了什么？"）
