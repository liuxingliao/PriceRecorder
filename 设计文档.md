### 2.2 核心模型说明

#### ProductRecord (商品记录)
- **用途**: 存储每次购买的商品记录
- **关键字段**:
  - `name`: 商品名称（必填）
  - `quantity` / `unit`: 数量和单位
  - `totalPrice` / `unitPrice`: 总价和单价（自动计算）
  - `merchantID`: 关联商家
  - `purchaseDate`: 购买时间
  - `receiptPhoto`: 小票照片（可选）
- **索引**: `id` (唯一), `createTime` (排序)

#### Merchant (商家)
- **用途**: 存储商家信息
- **关键字段**:
  - `name`: 商家名称（必填）
  - `categoryID`: 关联分类（可选）
  - `address` / `phone`: 联系信息
- **关系**: 一个商家可对应多个商品记录

#### MerchantCategory (商家分类)
- **用途**: 商家分类管理
- **关键字段**: `name` (分类名称)

#### Brand (品牌)
- **用途**: 品牌列表管理
- **关键字段**: `name` (品牌名称)

#### Receipt (小票)
- **用途**: 小票存档
- **关键字段**:
  - `merchantID`: 关联商家
  - `purchaseDate`: 购买时间
  - `photo`: 照片数据

### 3.3 OCR识别模块 (OCRService)

**架构设计**:
```
UIImage → VNRecognizeTextRequest → [OCRResult] → parseReceiptItems → [PendingProduct]
```

**核心流程**:
1. **文字识别**: 使用 Vision 框架的 `VNRecognizeTextRequest`
   - 识别语言: 中文简体 (zh-Hans) + 英文 (en-US)
   - 识别级别: `.accurate` (高精度)
   - 启用语言校正

2. **结果解析**: `parseReceiptItems(from:)`
   - 将识别的文本按行分割
   - 逐行解析商品信息

3. **单行解析**: `parseLine(_:)`
   - 正则匹配价格模式: `\d+[.,]\d{2}`
   - 提取商品名称和价格
   - 解析数量 (如 "2x 牛奶")

**正则表达式**:
```swift
// 价格匹配: 匹配 9.99 或 9,99 格式
let pricePattern = "\\d+[.,]\\d{2}"

// 数量匹配: 匹配 "2x"、"3×"、"5 " 等格式
let quantityPattern = "^(\\d+)\\s*[xX×]?\\s*"
```

### 3.4 搜索模块 (SearchView)

**两种搜索模式**:

1. **商品模式**
   - 列表项: 单条商品记录
   - 显示字段: 商品名、商家、时间、总价、数量、单价
   - 点击进入详情页

2. **商家模式**
   - 列表项: 按商家分组
   - 显示该商家该商品的最近一次记录
   - 点击显示该商家的历史记录

**排序选项**:
- 按时间 (默认): `purchaseDate` 降序
- 按价格: `totalPrice` 升序
- 按名称: `name` 字母序

**过滤逻辑**:
```swift
var filteredProducts: [ProductRecord] {
    var products = searchText.isEmpty ? allProducts : allProducts.filter {
        $0.name.localizedCaseInsensitiveContains(searchText)
    }

    switch sortOption {
    case .time: products.sort { $0.purchaseDate > $1.purchaseDate }
    case .price: products.sort { $0.totalPrice < $1.totalPrice }
    case .name: products.sort { $0.name < $1.name }
    }

    return products
}
```

### 3.5 比价模块 (PriceComparisonView)

**三步流程**:
```
1. 选择商品 → 2. 选择商家 (最多5个) → 3. 显示比价结果
```

**数据聚合**:
- 按商品名称分组去重
- 筛选出有该商品记录的商家
- 每个商家取最近100条记录

**图表展示** (SwiftCharts):
```swift
Chart {
    ForEach(comparisonData, id: \.merchant.id) { data in
        ForEach(data.records) { record in
            LineMark(
                x: .value("日期", record.purchaseDate),
                y: .value("单价", record.unitPrice)
            )
            .foregroundStyle(by: .value("商家", data.merchant.name))
        }
    }
}
```

**价格对比表**:
- 按最新单价升序排列
- 高亮显示最便宜的商家
- 显示购买时间

### 3.6 CSV导入导出模块 (CSVService)

**CSV格式定义**:
```
商品名称,品牌,数量,单位,规格,总价,单价,商家名称,商家分类,购买时间,备注,创建时间,更新时间
```

**导出流程**:
1. 查询所有商品记录
2. 关联商家和分类信息
3. 构造 `CSVExportData` 数组
4. 调用 `exportCSV(data:)` 生成CSV字符串
5. 通过 `fileExporter` 保存文件

**导入流程**:
1. 通过 `fileImporter` 选择文件
2. 读取文件内容
3. 调用 `parseCSV(_:)` 解析
4. 自动创建不存在的商家和分类
5. 插入商品记录
6. 显示导入结果统计

**CSV字段转义**:
- 字段包含 `,`、`"`、`\n` 时需要用双引号包裹
- 字段内的双引号需要转义为 `""`

```swift
private func escapeCSVField(_ field: String) -> String {
    if field.contains(",") || field.contains("\"") || field.contains("\n") {
        let escaped = field.replacingOccurrences(of: "\"", with: "\"\"")
        return "\"\(escaped)\""
    }
    return field
}
```

### 3.7 数据统计模块 (StatisticsView)

**统计项**:

| 统计项 | 计算方式 |
|-------|---------|
| 商品总数 | `products.count` |
| 商家数量 | `merchants.count` |
| 分类数量 | `categories.count` |
| 品牌数量 | `brands.count` |
| 总支出 | `products.reduce(0) { $0 + $1.totalPrice }` |
| 最贵商品 | `products.max { $0.unitPrice < $1.unitPrice }` |
| 最常购买 | 按商品名称分组统计次数 |
| 商家排行 | 按商品数量排序，取前5 |

## 8. 性能优化策略

### 8.1 查询优化

- 使用 `@Query` 的排序参数，避免内存排序
- 首页只取最近10条: `Array(allProducts.prefix(10))`
- 比价页只取最近100条: `.suffix(100)`

### 8.2 OCR性能

- 在后台队列执行识别: `DispatchQueue.global(qos: .userInitiated).async`
- 识别完成后切回主线程更新UI

### 8.3 列表性能

- 复用 SwiftUI List 的单元格
- 避免在 `ForEach` 中进行复杂计算
- 使用 `.listStyle(.plain)` 提升性能
