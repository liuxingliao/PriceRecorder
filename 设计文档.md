### 2.2 核心模型说明

#### ProductRecord (商品记录)
- **用途**: 存储每次购买的商品记录
- **关键字段**:
  - `name`: 商品名称（必填）
  - `quantity` / `unit`: 数量和单位
  - `totalPrice` / `unitPrice`: 总价和单价（自动计算）
  - `merchantID`: 关联商家
  - `purchaseDate`: 购买时间
  - `receiptPhoto`: 小票照片（可选）
- **索引**: `id` (唯一), `createTime` (排序)

#### Merchant (商家)
- **用途**: 存储商家信息
- **关键字段**:
  - `name`: 商家名称（必填）
  - `categoryID`: 关联分类（可选）
  - `address` / `phone`: 联系信息
- **关系**: 一个商家可对应多个商品记录

#### MerchantCategory (商家分类)
- **用途**: 商家分类管理
- **关键字段**: `name` (分类名称)

#### Receipt (小票)
- **用途**: 小票存档
- **关键字段**:
  - `merchantID`: 关联商家
  - `purchaseDate`: 购买时间
  - `photo`: 照片数据

#### APIConfig (API配置)
- **用途**: 存储豆包链接配置
- **关键字段**:
  - `doubaoLink`: 豆包链接地址

7. 保存

**JSON数据格式**:
```json
[
  {
    "name": "商品名称",

    "unit": "个",
    "totalPrice": 10.0,
    "spec": "规格（可选）"
```

**关键特性**:
- 支持批量录入商品
- 实时计算单价 (unitPrice = totalPrice / quantity)
- 支持商家快速新增
- 支持删除和编辑待录入商品

### 3.3 搜索模块 (SearchView)

**两种搜索模式**:

1. **商品模式**
   - 列表项: 单条商品记录
   - 显示字段: 商品名、商家、时间、总价、数量、单价
   - 点击进入详情页

2. **商家模式**
   - 列表项: 按商家分组
   - 显示该商家该商品的最近一次记录
   - 点击显示该商家的历史记录

**排序选项**:
- 按时间 (默认): `purchaseDate` 降序
- 按价格: `totalPrice` 升序
- 按名称: `name` 字母序

**过滤逻辑**:
```swift
var filteredProducts: [ProductRecord] {
    var products = searchText.isEmpty ? allProducts : allProducts.filter {
        $0.name.localizedCaseInsensitiveContains(searchText)
    }

    switch sortOption {
    case .time: products.sort { $0.purchaseDate > $1.purchaseDate }
    case .price: products.sort { $0.totalPrice < $1.totalPrice }
    case .name: products.sort { $0.name < $1.name }
    }

    return products
}
```

### 3.5 CSV导入导出模块 (CSVService)

**CSV格式定义**:
```
商品名称,品牌,数量,单位,规格,总价,单价,商家名称,商家分类,购买时间,备注,创建时间,更新时间
```

**导出流程**:
1. 查询所有商品记录
2. 关联商家和分类信息
3. 构造 `CSVExportData` 数组
4. 调用 `exportCSV(data:)` 生成CSV字符串
5. 通过 `fileExporter` 保存文件

**导入流程**:
1. 通过 `fileImporter` 选择文件
2. 读取文件内容
3. 调用 `parseCSV(_:)` 解析
4. 使用本地缓存避免重复创建商家和分类
5. 插入商品记录
6. 显示导入结果统计

**商家去重机制**:
- 导入前先将现有商家和分类加载到本地缓存字典
- 导入时使用缓存查找，避免重复创建
- 新创建的商家和分类也加入缓存

**CSV字段转义**:
- 字段包含 `,`、`"`、`\n` 时需要用双引号包裹
- 字段内的双引号需要转义为 `""`

```swift
private func escapeCSVField(_ field: String) -> String {
    if field.contains(",") || field.contains("\"") || field.contains("\n") {
        let escaped = field.replacingOccurrences(of: "\"", with: "\"\"")
        return "\"\(escaped)\""
    }
    return field
}
```

## 8. 性能优化策略

### 8.1 查询优化

- 使用 `@Query` 的排序参数，避免内存排序
- 首页只取最近10条: `Array(allProducts.prefix(10))`
- 比价页只取最近100条: `.suffix(100)`

### 8.2 v1.2 版本性能优化

#### SearchView 性能优化

**问题**: 原始实现存在 N+1 查询问题，每行都通过 `@Query` 查询商家数组并用 `first(where:)` 查找。

**解决方案**:
1. 添加 `merchantById` 字典缓存，将商家查找从 O(N) 优化到 O(1)
2. 修改 `SearchProductRow`，直接传入 `merchantName` 而非每行都查询
3. 消除 N+1 查询问题，大幅提升大数据量时的搜索性能

**核心代码**:
```swift
// 商家字典缓存
private var merchantById: [UUID: Merchant] {
    Dictionary(uniqueKeysWithValues: merchants.map { ($0.id, $0) })
}

// 直接传入 merchantName
SearchProductRow(product: product, merchantName: merchantName)
```

#### 输入验证系统

**新增功能**: 在商品录入时添加完整的输入验证，防止无效数据。

**验证内容**:
- 商品名称: 非空、≤100字符
- 数量: >0、≤1,000,000
- 总价: >0、≤1,000,000
- 单位: 非空、≤20字符
- 品牌/规格/备注: 长度限制

**实现方式**:
- 在 `ProductEntryView` 中集成 `ProductValidation` 验证工具
- 使用 `do-catch` 捕获验证错误
- 通过 Alert 显示友好的错误提示

#### OCRService 优化

**优化内容**:
1. **图片压缩**: 添加图片压缩功能，最大 1024px 边长，减少内存占用
2. **Async/Await**: 新增现代 `async/await` 版本的 API
3. **向后兼容**: 保留原有 completion handler 版本（标记为 deprecated）

### 8.2 列表性能

- 复用 SwiftUI List 的单元格
- 避免在 `ForEach` 中进行复杂计算
