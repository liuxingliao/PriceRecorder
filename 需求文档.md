## 1. 项目概述

### 1.1 项目背景
开发一款iOS应用，帮助用户记录不同商家的商品价格信息。用户可通过豆包智能录入快速录入商品，并可随时查看历史价格、比较不同商家的价格。v1.3版本新增AI咨询、数据备份、图表增强等功能。

### 1.2 核心功能
- 豆包智能录入 + 手动输入商品
- 商家管理（支持分类）
- 商品搜索和比价
- 历史价格趋势图（支持滑动、点击交互、时间范围切换）
- 数据备份和CSV导入导出
- 数据统计（增强版）
- AI咨询功能
- 完整数据备份（含图片，支持备份选项）
- 数据存储空间统计

## 2. 功能需求详细说明

### 2.1 首页 - 最近录入

**功能描述**: 显示用户最近录入的商品记录

**详细需求**:
- 显示最近录入的10条商品记录
- 每条记录一行，显示：商品名称、价格、数量、单位
- 点击商品进入详情页
- 页面顶部显示"商品录入"按钮
- 点击"商品录入"按钮弹出选择框：手动输入 / 豆包录入
- 空状态时显示提示信息和快速录入入口

---

### 2.2 商品录入功能

#### 2.2.1 录入方式选择
- 提供两种录入方式：手动输入、豆包录入
- 用户可自由切换选择

#### 2.2.2 豆包录入流程
1. **打开豆包**: 点击按钮打开配置的豆包链接
2. **复制数据**: 在豆包中复制商品JSON数据
3. **粘贴JSON**: 将JSON数据粘贴到编辑框
4. **解析商品**: 点击解析按钮，解析JSON数据
5. **确认编辑**:
   - 显示解析出的商品列表
   - 支持删除商品
   - 支持编辑商品详情
   - 点击商品进入详情编辑页
6. **保存确认**:
   - 选择商家（支持新增商家）
   - 确认/修改购买时间（默认当前时间）
   - 保存

#### 2.2.3 手动输入流程
- 支持一次输入多个商品
- 流程与豆包录入的编辑页类似
- 可随时添加/删除商品

#### 2.2.4 输入字段限制（v1.3新增）
所有商品录入页面（手动录入、豆包录入解析后编辑、商品详情编辑）都需遵守：
- **商品名称**：最多100字符，超出时禁止继续输入，实时显示已输入字数/最大字数
- **规格**：最多50字符，超出时禁止继续输入，实时显示已输入字数/最大字数
- **备注**：最多200字符，超出时禁止继续输入，实时显示已输入字数/最大字数

#### 2.2.5 照片质量选择（v1.3新增）
- 照片选择时提供质量选项：压缩图（默认）/原图
- 压缩图：系统默认压缩，节省存储空间
- 原图：完整质量，占用空间较大
- 用户选择后，下次默认使用上次的选择

#### 2.2.6 JSON数据格式

```json
[
  {
    "name": "商品名称",
    "quantity": 1.0,
    "unit": "个",
    "totalPrice": 10.0,
    "spec": "规格（可选）"
  }
]
```

### 2.3 搜索页面

**功能描述**: 搜索商品记录，支持多种查看方式

#### 2.3.1 AI咨询按钮（v1.3新增）
- 页面左上角添加"AI咨询"按钮
- 点击按钮弹出AI咨询弹窗
- 弹窗样式类似微信聊天窗口

#### 2.3.2 AI咨询功能（v1.3新增）

**弹窗界面结构**：
- 顶部：聊天记录区域（滚动显示，支持查看历史）
- 底部：输入区域
  - 左侧：历史记录按钮
  - 中间：文本输入框
  - 快捷按钮：插入"/商品咨询："
  - 右侧：发送按钮

**聊天功能**：
- 显示用户输入和AI回复的对话历史
- 支持上下滚动查看历史消息
- 消息样式区分：用户消息（右侧气泡）、AI回复（左侧气泡）
- 支持输入多行文本
- 对话信息保存在本地SwiftData中（删除App可丢失，数据备份时不含聊天记录）

**快捷按钮功能**：
- 点击"/商品咨询："按钮，在输入框中插入该文字
- 当输入框包含"/商品咨询："时，发送前自动查询本地商品数据

**商品咨询流程**：
1. 用户输入包含"/商品咨询："的文本
2. 点击发送后，App查询本地商品数据库
3. 将商品数据（包括名称、规格、价格、商家、购买时间等）整理成文本
4. 与用户的问题一起发送给大模型API
5. 大模型根据市场情况返回商品信息，包括：
   - 市场参考价格
   - 规格建议
   - 食品安全相关信息（如有）
   - 价格趋势分析
6. 在聊天区域显示AI的回复

**历史记录功能**：
- 点击历史记录按钮，显示历史对话列表
- 支持快速查看之前的问答
- 历史记录保存在本地

#### 2.3.3 搜索模式切换
- 顶部搜索框 + 切换按钮
- 两种模式：商品模式 / 商家模式

#### 2.3.4 商品模式
- 每行显示：商品名称、商家、时间、总价、数量、单价
- 点击进入商品详情页
- 支持修改数据

#### 2.3.5 商家模式
- 每行显示：商家名称、该商品最近一次价格、时间、总价、数量、单价
- 点击显示该商家该商品的所有历史记录

#### 2.3.6 排序功能
支持以下排序方式：
- 按时间
- 按价格
- 按商品/商家名称首字母

### 2.4 比价功能

**功能描述**: 比较同一商品在不同商家的价格

#### 2.4.1 比价流程
1. **选择商品**: 从商品列表中选择一个商品
2. **选择商家**: 最多选择5个商家进行比较
3. **显示结果**:
   - 历史价格趋势图（显示最近100次数据）
   - 最新价格对比表
   - 标识出最便宜的商家

#### 2.4.2 图表交互增强（v1.3新增）
- **左右滑动**: 支持左右滑动查看更多历史数据
- **点击/长按交互**: 点击或长按图表上的数据点时，显示详情信息：
  - 价格
  - 购买时间
  - 商家（如果有多条线）
- 显示的信息使用气泡样式
- 再次点击或点击其他位置关闭详情

#### 2.4.3 图表时间范围切换（v1.3新增）
- 在图表标题旁边添加时间范围选择器（分段控件）
- 支持以下时间范围：
  - 日：显示最近7天数据
  - 周：显示最近14天数据
  - 月：显示最近30天数据（默认）
  - 年：显示最近365天数据
- 切换时间范围时图表自动更新显示数据

### 2.5 商品详情页

**功能描述**: 显示商品的完整信息并支持编辑

**显示字段**:
- 商品名称
- 品牌
- 规格
- 数量、单位
- 总价、单价
- 商家
- 购买时间
- 小票照片（如有）
- 备注

**功能**:
- 支持编辑所有字段（字段长度限制同录入页面）
- 支持添加/更换小票照片
- 照片选择时支持质量选择（压缩图/原图）
- 单价修改后不更新总价

### 2.6 设置页面

#### 2.6.1 商家管理
- 商家列表显示
- 新增/编辑/删除商家
- 商家信息：名称、分类、地址、电话、备注
- 支持商家分类管理（用户自由添加分类）

#### 2.6.2 豆包配置
- 豆包链接配置
- 支持自定义豆包链接

#### 2.6.3 大模型配置（v1.3新增）
- 新增"大模型配置"入口
- 配置项包括：
  - 提供商选择（豆包、OpenAI、Claude、通义千问、文心一言、自定义）
  - API Key（安全输入）
  - API端点
  - 模型名称
  - 温度参数
  - 最大Token数
  - 启用/禁用开关
- 配置保存到SwiftData
- 支持配置多个提供商，同一时间只启用一个
- 列表显示已有配置，支持编辑/删除
- 编辑界面右上角保存按钮和底部保存按钮

#### 2.6.4 接口测试功能（v1.3新增）
- 在大模型配置编辑页面添加"测试接口"按钮
- 按钮位置在测试连接区域
- 点击后使用当前填写的配置信息发送测试请求
- 测试消息："你好，请回复'测试成功'"
- 显示测试状态：加载中/成功/失败
- 测试成功显示绿色提示，失败显示红色错误信息
- 必填项未填写时按钮禁用

#### 2.6.5 数据导入导出
- **CSV导出**: 导出全部数据，一行一条，包含所有字段
- **CSV导入**: 仅支持本应用导出的CSV格式，支持商家去重
- 导入结果显示（成功/失败数量）

#### 2.6.6 iCloud备份
- 手动备份功能（隐藏在调试配置中）
- 自动备份开关（隐藏在调试配置中）
- 显示上次同步时间
- 全量备份

#### 2.6.7 数据备份（v1.3新增）
- 新增"数据备份"入口
- 完整备份功能，包括：
  - 所有商品记录
  - 所有商家信息
  - 所有小票照片
  - **不含AI聊天记录**
- 备份方式：
  - 导出为JSON格式，照片数据使用Base64编码
  - 可选择保存位置（文件App）
- 数据恢复功能：
  - 从JSON文件恢复数据
  - 恢复前确认备份信息（类型、商品数量、是否有照片）
  - 支持选择恢复内容
  - 显示恢复进度和结果

#### 2.6.8 备份选项（v1.3新增）
- 创建备份时支持选择备份内容：
  - 全部备份：备份所有数据和照片（文件最大）
  - 仅备份数据：只备份商品、商家等数据（不含照片）
  - 仅备份照片：只备份照片数据
- 恢复时支持选择恢复内容：
  - 根据备份文件内容动态显示可选选项
  - 全部备份文件支持：全部恢复、仅恢复数据、仅恢复照片
  - 仅数据备份文件只支持：仅恢复数据
  - 仅照片备份文件只支持：仅恢复照片

#### 2.6.9 数据统计（增强v1.3）

**基础统计信息**：
- 商品总数
- 商家数量
- 分类数量
- 总支出金额
- 最贵商品
- 最常购买商品

**存储空间统计（v1.3新增）**：
- 数据总占用空间
- 照片占用空间
- 其他数据占用空间
- 以MB/GB为单位显示

**商家商品信息（增强v1.3）**：
- 显示所有商家列表（不再只显示前5）
- 列表项显示：商家名称、商品数量、消费总额
- 按消费总额倒序排列
- 点击商家后弹出详情弹窗

**商家详情弹窗（v1.3新增）**：
弹窗显示该商家的详细信息：
- 商家名称
- 消费总额（大号字体突出显示）
- 商品总数量
- 每个种类商品的数量列表（按数量倒序排列）
  - 示例：
    - 牛奶：15件
    - 面包：10件
    - 鸡蛋：8件
- 弹窗支持关闭

#### 2.6.10 调试功能
- 清空所有数据按钮（需二次确认）
- 大模型Debug开关（v1.3新增）：
  - 开启后记录所有大模型API调用情况
  - 显示调用历史
  - 显示请求参数和响应数据
  - 显示调用时间、耗时
  - 调试记录保存在内存中

## 5. 技术要求

- **数据持久化**: SwiftData

### 5.3 CSV导出格式
```

```
